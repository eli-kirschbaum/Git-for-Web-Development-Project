1. What is Semantic HTML?
    - HTML tags with meaning
2. What is HTML used for? 
    - to display data in a graphical way/to make the structure of a web page
3. What is an attribute and where do we put it?
    - an attribute is a way to provide additional info for html elements, you put it with the tag
4. What is the h1 tag used for? How many times should I use it on a page?
    - h1 tag is used for the most important/most visible phrase/word/title on your web page, it is only used once.
5. Name two tags that have required attributes
    -  <img>  <link>
6. What do we put in the head of our HTML document? 
    - info about your web page that isnt displayed on the page
7. What is an id? 
    - an attribute to label an element with a unique ID that can only be used once.
8. What elements can I add an id to? 
    - any html element
9. How many times can I use the same id on a page?
    - once
10. What is a class? 
    - an attribute to name an element
11. What elements can I add a class to?
    - any html element
12. How many times can I use the same class on a page? 
    - as many times as you want
13. How do I get my link to open in a new tab?
    - include a target attribute of _blank. example: <a href:"...." target="_blank"> ... </a>
14. What is the alt attribute in the image tag used for?
    - provides different text to be displayed in case the image cant be displayed
15. How do I reference an id?
    - with a #.   example:   #someID { .... }
16. What is the difference between a section and a div
    - a div does not have any meaning
17. What is CSS used for? 
    - to style and present web pages 
18. How to we select an element? Example - every h2 on the page
    - h2 {...}
19. What is the difference between a class and an id? - Give me an example of when I might use each one
    - an id can only be used once.  example: ID used for headers, footers, etc or things you only use once per page.  Class used for elements that appear multiple times like paragraphs or divs.
20. How do we select classes in CSS?
    - with a .   example:   .someClass {...}
21. How do we select a p element with a single class of “human””?
    - p .human {...}
22. What is a parent child selector? When would this be useful? 
    - a selector for elements within other elements, as in paragraphs inside of sections. Useful for editing only select parts of your page.
23. How do you select all links within a div with the class of sidebar?
    - .sidebar div {...}
24. What is a pseudo selector?
    - a selector that is for only specific states of an element
25. What do we use the change the spacing between lines?
    - ... {
        line-height: ...
    }
26. What do we use to change the spacing between letters?
    - ... {
        letter-spacing: ...
    }
27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
    - ... {
        text-transform: uppercase
        }
    - ... {
        text-transform: lowercase
    }
    - ... {
        text-transform: capitalize
    }
28. How do I add a 1px border around my div that is dotted and black?
    - div {
        border-style: dotted;
        border-width: 1px;
    }
29. How do I select everything on the page? 
    - * {...}
30. How do I write a comment in CSS?
    - /*  ... */
31. How do I find out what file I am in, when I am using the command line? 
    - $ pwd
32. Using the command line - how do I see a list of files/folders in my current folder?
    - $ ls
33. How do I remove a file via the command line? Why do I have to be careful with this? 
    - $ rm <file name>    
34. Why should I use version control? 
35. How often should I commit to github?
    - every 20 min
36. What is the command we would use to push our repo up to github? 
    - git push -u origin branch-name
37. Walk me through Lambda's git flow.
    1) Fork the repo on github
    2) clone the repo by copying the the code link on github then using git clone in the terminal
    3) change the directory to that repo by using cd repo name
    4) create a branch by typing git checkout -b firstname-lastname in the terminal
    5) open the code in VS code with code . 
    6) when you have changes made (every 20 mins) push the code to github 
        a) first use git add . which puts the changes in a staging area
        b) commit the changes with a clear message describing the changes with git commit -m 'message'
        c) push the changes to your created branch with git push -u origin branch-name
    7) create a pull request by clicking Compare & Pull Request in github
    8) choose your repo as the base
    9) choose the main repo as the comparison
    10) copy the url for the pull request and post it in canvas
